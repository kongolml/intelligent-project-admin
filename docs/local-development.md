# Backend Local Development

## Prerequisites

- Node.js 20 (via nvm recommended)
- MongoDB instance (local or remote, e.g., MongoDB Atlas)
- DigitalOcean Spaces bucket (for media storage)

## Setup

```bash
# Clone the repo
git clone <repo-url>
cd intelligent-project-adm

# Install dependencies
npm install
```

## Environment Variables

Create `.env` from the example:

```bash
cp .env.example .env
```

Fill in the required values:

```env
# MongoDB connection string
DATABASE_URL=mongodb://localhost:27017/intelligent-project

# Payload encryption secret (any random string)
PAYLOAD_SECRET=your-random-secret-here

# DigitalOcean Spaces credentials
DIGITALOCEAN_SPACE_REGION=fra1
DIGITALOCEAN_SPACE_ENDPOINT=https://fra1.digitaloceanspaces.com
DIGITALOCEAN_SPACE_BUCKET=your-bucket-name
DIGITALOCEAN_SPACE_HOST=fra1.digitaloceanspaces.com
S3_ACCESS_KEY=your-access-key
S3_SECRET_KEY=your-secret-key

# URLs
NEXT_PUBLIC_SERVER_URL=http://localhost:3000
FRONTEND_URL=http://localhost:3001

# Webhook secret (must match frontend's PAYLOAD_WEBHOOK_SECRET)
PAYLOAD_WEBHOOK_SECRET=your-webhook-secret
```

## Running

```bash
# Start dev server
npm run dev
```

The CMS runs at `http://localhost:3000`:
- Admin panel: `http://localhost:3000/admin`
- REST API: `http://localhost:3000/api/<collection-slug>`

On first run, you'll be prompted to create an admin user.

### Clean Start

If you encounter issues with cached state:

```bash
npm run devsafe   # deletes .next cache, then starts dev
```

## Commands

| Command | Description |
|---------|-------------|
| `npm run dev` | Start dev server |
| `npm run devsafe` | Clean `.next` cache + start dev |
| `npm run build` | Production build |
| `npm run start` | Start production server |
| `npm run lint` | ESLint |
| `npm run generate:types` | Regenerate `src/payload-types.ts` from collections |
| `npm run generate:importmap` | Regenerate Payload admin import map |
| `npm run migrate:wp` | Import data from WordPress |
| `npm run fix:image-connections` | Repair media relationships post-migration |

## Adding a New Collection

1. Create the collection config in `src/collections/YourCollection.ts`
2. Add it to the `collections` array in `src/payload.config.ts`
3. Run `npm run generate:types` to update TypeScript types
4. Run `npm run generate:importmap` to update the admin import map

## Notes

- `src/payload-types.ts` is auto-generated and in `.gitignore` — never edit it manually
- Files in `src/app/(payload)/` are auto-generated by Payload — do not modify
- The project uses ESM (`"type": "module"` in package.json)
- Rich text fields use the Lexical editor with auto-derived HTML fields via `lexicalHTMLField`
